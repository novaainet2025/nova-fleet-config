# GS인증 문서 작성 (UM/PD)

> 근거: 실제 GS인증 결함보고서(D001~D007 문서결함 + P0001~P0017 제품결함)에서 역산한 "심사관이 실제 지적한" 체크포인트 + 검증된 제출문서 구조. ★발명/가상코드 금지 — 모든 내용은 실제 SW 동작 기준으로만 작성★ (피어보고는 T4, 소스 grep로 교차검증).

## 0. 배경
- GS(Good Software)인증 = 상용SW 품질 인증. 기반표준 **KS X ISO/IEC 25051**(품질요구·시험) + 25010 품질모델.
- 핵심 제출문서 2종: **사용자취급설명서(UM)**, **제품설명서(PD)**.
- 심사 본질 = "문서 ↔ 실제 SW 동작 일치성" + 기능적합성·보안·사용성.

## 1. 작성 규칙
### 1-1. 필수 구조 (검증된 TOC)
- **UM**: 1.제품 개요(1.1개요 · 1.2용도및목적 · 1.3제품·공급자정보 · 1.4운영환경 · 1.5설명서 구성및제공 · 1.6주요 제공 작업및서비스 · 1.7제한사항및성능) / 2.제품 사용설명(2.1설치방법 · 2.2기능목록 · 2.3사용방법 · 2.4백업및복구 · 2.5로그 · 2.6외부연동) / 3.고객지원 · 4.용어및약어 · 5.색인
- **PD**: 유사 구조 + 2.4 성능 · 2.6 보안요구사항 강조.
### 1-2. 버전/날짜 형식
- 제품명: `<정식 제품명> v1.0` (전 문서 버전 일관).
- 개정이력 표: 개정번호 `Rev.0` / 일자 `YYYY.MM.DD` / 내용 `최초 작성` / 작성자.
### 1-3. 금지/주의 표현
- ❌ 모호한 결과 메시지 단독("확인"/"완료"/"확인되었습니다") → ✅ 수행기능·주체·결과 명시 (P0006).
- ❌ 인증범위 외 기능 단정("AI") → ✅ 검증 가능한 표현으로 범위정리("키워드·규칙 기반") (D001/D003).
- ❌ 문서-코드 불일치 → ✅ 입력 유효범위·에러메시지는 **코드 실제 문구와 100% 동일** (P0011/P0012).

## 2. 요령/팁 — 심사관 지적 포인트 (D/P = 실제 지적사례)
### 2-1. 문서결함(D) 빈출
- 1.1 개요에 **사용환경·사용자 대상·제공 기능** 구체화 (D001/D003).
- **운영환경 구성도 ↔ 운영환경 표 일치**(구성요소·캡션 대응) (D002/D004).
- **2.3 메뉴별 상세 사용법 ↔ 2.2 기능목록 정합**(1:1) (D005).
- ★기능설명 필수 5요소(모든 기능)★: ①개요·목적·용도 ②입력필드 유효범위 표 ③메시지(에러·상태) 표 ④따라하기 절차(번호 단계) (D006). ※읽기전용 화면은 입력/에러표 "해당없음" 명시.
- 권한별(RBAC) 역할·접근환경 설명 (D007).
- 보안·에러처리·성능·제한사항 항목 누락 금지.
### 2-2. 제품결함(P) = 기능/보안 품질 (문서+코드 동시충족)
- 데모/사용예제(P0001) · 오류메시지 원인+해결(P0002) · 사용자 설정기능(P0003) · 키보드 접근성 aria/tab(P0004) · 동작 모니터링(P0005) · 메시지 구체성(P0006) · 실행취소·재확인 다이얼로그(P0007) · 입력 유효성 거부+안내+글자수제한(P0008) · 제품식별정보·버전(P0009).
- 보안: 암호알고리즘 명시(P0010: bcrypt cost12 / JWT HS256 / SRTP / TLS1.2+) · 아이디 5자↑+예약어금지(P0011) · 비번 9자↑+3종조합(P0012) · SQLi prepared(P0013) · XSS 자동이스케이프(P0014) · 비번 단방향해시(P0015) · 개인영상 미수집/암호화(P0016) · 미사용 10분 세션종료(P0017).

## 3. 워크플로우 (초안→제출)
1. **Ground truth 확보**: 실제 코드/실행화면 기준. ★발명·범용어댑터·가상코드 금지★ (소스 grep + 앱 실캡처).
2. **구조 템플릿**(위 TOC)으로 초안.
3. **기능별 5요소** 채우기 + 실행화면 캡처 삽입(실캡처/Higgsfield 정제, md5 고유검증).
4. **코드 연동**: 문서의 규칙(유효범위·에러메시지)=코드 실제값과 동기화. ★코드 수정 시 문서 동시갱신★(예: 코드 9자 → 문서도 9자).
5. **HWPX 작성**: md→hwpx 빌더 또는 section0.xml surgical 편집(clone+linesegarray 비움+id 재할당). 특수문자 경로 한글 COM SAVE 함정 회피(zip 재패키징).
6. **자가 셀프오딧**: 결함보고서 D/P 체크리스트로 사전 점검.
7. **시각검증**: rhwp SVG→Chrome PNG로 삽입문단 겹침/클리핑 확인 + ★한글 실오픈 1회 육안★(rhwp≈한글 근사). find_text는 레이아웃 무검증.
8. **제출** + 결함 대응 시 증거(캡처/grep/렌더) 첨부.
